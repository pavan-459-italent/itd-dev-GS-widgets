# AGENTS.md

## Project overview

This repository is a template for building and publishing widgets with optional connectors, global scripts, and global stylesheets. Widgets are defined in `widgets/<widget_name>/` with
`widget.json` and an entry file at `dist/content.html`. Connectors are defined per-widget in `connectors.json` alongside `widget.json`. Scripts and stylesheets are defined globally in `scripts/` and `stylesheets/` directories. The service pulls only the directory at `source.path` (e.g. `dist/`). The widget registry is generated into `widget_registry.json` and the connector registry into `connectors_registry.json`.

## Setup commands

- Install dependency (required): `jq`
  - macOS: `brew install jq`
  - Linux: `sudo apt-get install jq` or `sudo yum install jq`

## Common commands

- Build registry: `./bin/build-registry.sh`
- Dry run (no file write): `./bin/build-registry.sh --dry-run`
- Validate existing registry: `./bin/build-registry.sh --validate`
- Validate widget JSON syntax: `jq . widgets/<widget_name>/widget.json`
- Validate connector JSON syntax: `jq . widgets/<widget_name>/connectors.json`

## Repository structure

```
.
├── bin/build-registry.sh        # Defaults at top of script; no external config file
├── widget_registry.json         # Generated by build script (widgets, scripts, stylesheets)
├── connectors_registry.json     # Generated by build script from per-widget connectors.json files
├── scripts/
│   └── <script_name>/
│       ├── script.json          # Metadata: name, description, placement, rules
│       └── script.js            # JavaScript file (default name; override with contentFile)
├── stylesheets/
│   └── <stylesheet_name>/
│       ├── stylesheet.json      # Metadata: name, description, rules
│       └── style.css            # CSS file (default name; override with contentFile)
└── widgets/
    ├── WIDGET_SETUP.md
    ├── CONNECTOR_SETUP.md
    └── <widget_name>/
        ├── widget.json
        ├── connectors.json      # Optional: connector definitions for this widget
        └── dist/
            └── content.html     # Entry file; source.path points at dist so only it is pulled
```

## Reference examples

- **Simple widget**: `widgets/demo_widget/` — Minimal HTML widget
- **Configurable widget**: `widgets/card_grid/` — Configuration with multiple property types, includes connectors
- **React widget**: `widgets/react_hello_world/` — Build tool integration with Vite
- **Global script**: `scripts/analytics/` — Page-level JavaScript (analytics tracking)
- **Global stylesheet**: `stylesheets/theme/` — Shared CSS with design tokens
